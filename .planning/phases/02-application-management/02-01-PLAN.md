<task type="auto">
  <name>Task 3: Server Actions for application mutations</name>
  <files>lib/actions/applications.ts</files>
  <action>
Create `lib/actions/applications.ts` with `"use server"` directive at top.

Define `ActionState` type:
```typescript
export type ActionState = {
  success: boolean;
  errors?: Record<string, string[]>;
  message?: string;
};
```

Implement three Server Actions:

1. `createApplicationAction(prevState: ActionState, formData: FormData): Promise<ActionState>`
   - Extract fields from formData: name, testUrl, testUsername, testPassword
   - Validate with `applicationSchema.safeParse()`
   - On validation failure: return `{ success: false, errors: validatedFields.error.flatten().fieldErrors }`
   - On success: call `createApplication()` from service module
   - Call `revalidatePath("/applications")` and `revalidatePath("/")` (dashboard shows app count)
   - Call `redirect("/applications")` OUTSIDE try/catch (redirect throws NEXT_REDIRECT)
   - Structure: try { await createApplication(data) } catch { return error } then revalidatePath + redirect

2. `updateApplicationAction(id: string, prevState: ActionState, formData: FormData): Promise<ActionState>`
   - Same validation pattern as create
   - Call `updateApplication(id, data)` from service module
   - Wrap in try/catch for Prisma P2025 (record not found) errors
   - Call `revalidatePath("/applications")` and `revalidatePath("/")`
   - Return `{ success: true }` (no redirect -- modal closes on success)

3. `deleteApplicationAction(id: string): Promise<ActionState>`
   - Call `deleteApplication(id)` from service module
   - Wrap in try/catch for P2025 errors
   - Call `revalidatePath("/applications")` and `revalidatePath("/")`
   - Return `{ success: true }`

**Important patterns:**
- `redirect()` MUST be called outside try/catch (it throws a special error)
- Always `revalidatePath` before `redirect`
- Use the service module -- NEVER call Prisma directly
- Bind `id` parameter for update/delete using `.bind(null, id)` in the UI layer
