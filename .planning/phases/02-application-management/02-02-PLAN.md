Update `app/globals.css` to add custom CSS variables for the TestForge purple/blue theme. Keep the existing `@import "tailwindcss"` line.

Add a `@theme` block (Tailwind CSS 4 syntax) with custom color variables:

Define a purple/blue palette:
- Primary: A vibrant purple (e.g., `#7C3AED` / violet-600 range)
- Primary hover: Slightly darker (e.g., `#6D28D9`)
- Primary light: For backgrounds (e.g., `#EDE9FE`)
- Accent: A complementary blue (e.g., `#3B82F6`)
- Accent light: For subtle highlights (e.g., `#DBEAFE`)

Use Tailwind CSS 4 `@theme` syntax:
```css
@theme {
  --color-primary: #7C3AED;
  --color-primary-hover: #6D28D9;
  --color-primary-light: #EDE9FE;
  --color-accent: #3B82F6;
  --color-accent-light: #DBEAFE;
}
```

This makes these colors available as Tailwind utilities: `bg-primary`, `text-primary`, `border-accent`, etc.

Also add base body styles for the light theme:
```css
body {
  @apply bg-gray-50 text-gray-900;
}
```

Per user decision: Light theme with purple/blue accents, modern SaaS feel, differentiate from green/teal LogAnalyzer tool.
