**Create `app/applications/components/ApplicationModal.tsx`:**

Client component (`"use client"`) for both create and edit operations.

Props:
- `mode: "create" | "edit"`
- `application?: ApplicationView` (pre-filled data for edit mode)
- `isOpen: boolean`
- `onClose: () => void`

Implementation:

1. Use React 19 `useActionState` for form state management:
   - For create: `useActionState(createApplicationAction, initialState)`
   - For edit: `useActionState(updateApplicationAction.bind(null, application.id), initialState)`

2. Form fields (per user decision -- current schema only, no new columns):
   - **Name**: text input, required
   - **Test URL**: url input, required
   - **Test Username**: text input, required (per user decision: username visible by default)
   - **Test Password**: PasswordField component with reveal toggle, required

3. For edit mode: pre-fill all fields with existing data (per user decision: edit form pre-fills with decrypted values). The `application` prop contains already-decrypted credentials from `getApplication()`.

4. Validation errors: display below each field using `state.errors?.fieldName` (per user decision: inline errors below fields, red text).

5. Submit button: shows "Creating..." / "Updating..." while pending (use `pending` from useActionState).

6. General error message: display `state.message` if present (for server errors).

7. Modal overlay: semi-transparent dark backdrop, centered white modal card, close button (X), clicking backdrop closes modal.

8. On successful create: the Server Action redirects to /applications (handled by the action). On successful edit: close the modal and the list refreshes via revalidatePath.

9. When modal opens for edit, fetch the full application data (with decrypted credentials) via a server-side fetch. The page component should call `getApplication(id)` and pass the result to the modal.

**IMPORTANT PATTERN for edit pre-fill:**
Since `getApplication` is a server function, the applications page needs to handle fetching decrypted data. Options:
- Create a server action that returns the application data (not a mutation, just a read)
- Or fetch the data when the user clicks Edit via a separate server action

Recommended: Create a `getApplicationAction(id: string)` in `lib/actions/applications.ts` that calls `getApplication(id)` and returns the data. The modal's parent component can call this when edit is triggered.

Alternatively, the simpler approach: Make the applications list page smart enough to load application details on edit click. Use React state in the client wrapper to track which application is being edited, and use a `useEffect` or `startTransition` to fetch the data.
