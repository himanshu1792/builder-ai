  <behavior>
    ## URL Parsing
    - parseGitHubUrl("https://github.com/org/repo") -> { owner: "org", repo: "repo" }
    - parseGitHubUrl("https://github.com/org/repo.git") -> { owner: "org", repo: "repo" } (strips .git)
    - parseGitHubUrl("https://github.com/org") -> throws Error (missing repo segment)
    - parseAdoUrl("https://dev.azure.com/org/project/_git/repo") -> { project: "project", repoName: "repo" }
    - parseAdoUrl("https://org.visualstudio.com/project/_git/repo") -> { project: "project", repoName: "repo" } (legacy format)
    - parseAdoUrl("https://dev.azure.com/org/project") -> throws Error (missing _git segment)
    - extractRepoName("https://github.com/org/repo", "github") -> "org/repo"
    - extractRepoName("https://dev.azure.com/org/proj/_git/repo", "ado") -> "proj/repo"

    ## Slugify
    - slugify("My Web App") -> "my-web-app"
    - slugify("Test App (v2)") -> "test-app-v2"
    - slugify("  Hello  World  ") -> "hello-world"
    - slugify("App---Name") -> "app-name"

    ## PAT Validation (mocked fetch)
    - validateGitHubPat with 200 response + "repo" in X-OAuth-Scopes -> { valid: true }
    - validateGitHubPat with 401 response -> { valid: false, error: contains "Invalid PAT" }
    - validateGitHubPat with 404 + no "repo" scope -> { valid: false, error: contains "repo scope" }
    - validateGitHubPat with 404 + "repo" scope present -> { valid: false, error: contains "not found" }
    - validateGitHubPat with 200 but missing "repo" scope -> { valid: false, error: contains "repo scope" }
    - validateAdoPat with 200 response -> { valid: true }
    - validateAdoPat with 401 response -> { valid: false, error: contains "Invalid PAT" }
    - validateAdoPat with 403 response -> { valid: false, error: contains "Code (Read & Write)" }
    - validateAdoPat with 404 response -> { valid: false, error: contains "not found" }

    ## Repository CRUD (mocked Prisma)
    - createRepository encrypts PAT before storing
    - listRepositories(applicationId) returns repos for the given app
    - getRepository(id) decrypts PAT on retrieval
    - updateRepository(id, { outputFolder }) updates specified fields, encrypts PAT if provided
    - deleteRepository(id) removes the record

    ## Zod Schema
    - Valid GitHub input passes validation
    - Valid ADO input with organization passes validation
    - ADO input without organization fails with "Organization name is required"
    - Invalid URL format for provider fails with URL-specific error
    - Empty PAT fails with "Personal Access Token is required"

    ## Step 1: Prisma Schema + Migration

    Add Repository model to prisma/schema.prisma:

    ```prisma
    model Application {
      id            String       @id @default(cuid())
      name          String
      testUrl       String
      testUsername   String
      testPassword   String
      createdAt     DateTime     @default(now())
      updatedAt     DateTime     @updatedAt
      repositories  Repository[]
    }

    model Repository {
      id             String      @id @default(cuid())
      provider       String      // "github" or "ado"
      repoUrl        String
      pat            String      // Encrypted via lib/encryption.ts (AES-256-GCM)
      organization   String?     // Required for ADO, null for GitHub
      outputFolder   String      // e.g., "tests/my-web-app"
      applicationId  String
      application    Application @relation(fields: [applicationId], references: [id], onDelete: Cascade)
      createdAt      DateTime    @default(now())
      updatedAt      DateTime    @updatedAt
    }
    ```

    Run migration: `npx prisma migrate dev --name add_repository`
    Run generate: `npx prisma generate`

    ## Step 3: Implementation (GREEN phase)

    Create lib/repositories.ts with:
    - URL parsing functions (parseGitHubUrl, parseAdoUrl, extractRepoName)
    - slugify function
    - PAT validation functions (validateGitHubPat, validateAdoPat) using native fetch
    - CRUD functions: createRepository, listRepositories, getRepository, updateRepository, deleteRepository
    - All PAT fields encrypted on write, decrypted on read (same pattern as applications.ts)

    Create lib/schemas/repository.ts with:
    - repositorySchema: Zod object with provider enum, repoUrl URL validation, pat min(1), optional organization, outputFolder min(1)
    - Refinements: URL format validation per provider, organization required for ADO
    - Export RepositoryInput type

    Create lib/actions/repositories.ts with:
    - connectRepositoryAction(applicationId, prevState, formData): Zod validate -> PAT validate -> createRepository -> revalidatePath
    - updateRepositoryAction(repoId, prevState, formData): Zod validate partial -> updateRepository -> revalidatePath
    - deleteRepositoryAction(repoId, applicationId): deleteRepository -> revalidatePath
    - Import ActionState from lib/actions/applications.ts (reuse type)
    - revalidatePath for /applications/[id], /applications, and /

    ## Key Implementation Notes
    - GitHub PAT validation: GET https://api.github.com/repos/{owner}/{repo} with Bearer token. Check X-OAuth-Scopes header for "repo" scope. On 404, check if missing scope vs genuinely missing repo.
    - ADO PAT validation: GET https://dev.azure.com/{org}/{project}/_apis/git/repositories/{repoName}?api-version=7.1 with Basic auth (base64 of ":{pat}" -- note leading colon).
    - Support both ADO URL formats: dev.azure.com/{org}/... and {org}.visualstudio.com/...
    - Fine-grained GitHub PATs may not return X-OAuth-Scopes; treat 200 response as sufficient validation per research recommendation.
    - For updateRepositoryAction, only encrypt PAT if a new PAT value is provided (non-empty). If empty, skip PAT update.
